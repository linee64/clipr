# План разработки автономного агента Clipr AI
> [!NOTE]
> Идея функции: пользователь делает запрос вида «Создай мне N видеороликов на тему X и выкладывай их каждые Y часов в мои соцсети». Агент автоматически проходит цикл от генерации контент-плана до поэтапного постинга с учетом лимитов тарифа.

---

## 1. Архитектура и пользовательский сценарий (User Flow)
1. **Запуск агента (Ввод)**:
   - Пользователь вводит тему/цель агента (например, *«Лайфхаки на Python»*).
   - Выбирает платформы для публикации (сейчас: **X (Twitter)** и **LinkedIn**).
   - Задает общее количество видео (N) и интервал публикации (например, каждые 6 часов).
   - **Опция ИИ-озвучки**: Переключатель (Toggle) «Использовать закадровый ИИ-голос» (да/нет).
   - Выбирает шаблон стиля (шрифты, цветовые схемы, музыка).
2. **Контроль лимитов тарифа (Billing limits)**:
   - Система автоматически сверяет запрашиваемое количество видео $N$ с оставшимся балансом пользователя по тарифу (Free: 10/мес, Pro: 20/мес). 
   - Если лимитов не хватает, выводится предложение дозаказать лимиты или сделать апгрейд.
3. **Пакетная генерация и рендеринг (Backend)**:
   - Gemini генерирует контент-план из $N$ индивидуальных тем/сценариев по запросу пользователя.
   - Запускается последовательная очередь рендеринга (один за другим), чтобы не создавать пиковую нагрузку на CPU сервера.
   - Для каждого видео опционально генерируется озвучка ElevenLabs (если включена пользователем), подбирается b-roll в Pexels и накладываются субтитры.
4. **Автоматический постинг**:
   - Видео добавляются в таблицу расписания (`schedules`) со сдвигом во времени публикации на выбранный интервал (например, +6ч, +12ч, +18ч и т.д.).

---

## 2. Пошаговый план реализации

### Этап 1. База данных и логика лимитов (Backend)
- [ ] **Таблица `agent_batch_jobs`**:
  - Создать миграцию для новой таблицы: id, user_email, prompt, status, total_videos, interval_hours, use_voiceover, created_at.
- [ ] **Проверка лимитов на этапе запуска**:
  - Внедрить валидацию в `backend/routers/video.py`: проверять текущее количество доступных рендеров в Supabase `accounts` перед стартом батч-задачи. Блокировать запуск, если баланс меньше запрашиваемых видео.

### Этап 2. Последовательная очередь рендеринга (Task Queue)
- [ ] **Последовательный воркер (Sequential Queue)**:
  - Вместо параллельного рендеринга (который перегружает CPU) настроить очередь задач в `backend/workers/render.py` на запуск строго по очереди (FIFO, concurrency = 1).
  - При завершении рендеринга очередного видео обновлять прогресс задачи в БД.

### Этап 3. ИИ-генерация сценариев и озвучка (Gemini & ElevenLabs)
- [ ] **Адаптивный промпт для Gemini AI**:
  - Научить Gemini принимать произвольное число видео $N$ на вход и выдавать структурированный JSON с $N$ раскадровками.
- [ ] **Условный рендеринг озвучки**:
  - В `render.py` проверять флаг `use_voiceover`. Если флаг выключен, генерировать видео только с фоновой музыкой и субтитрами, пропуская шаг с ElevenLabs.

### Этап 4. Интерфейс управления агентом и автопостинг (Next.js Frontend)
- [ ] **Настройка платформ**:
  - Фокус на подключении **X (Twitter)** и **LinkedIn**. Исключить Instagram Reels из интерфейса агента до прохождения Meta App Review.
- [ ] **Экран конфигурации агента**:
  - Добавить чекбокс/тоггл для ИИ-озвучки.
  - Поле ввода для количества видеороликов (с динамической сверкой с лимитами аккаунта).
  - Выбор интервала (ввод любого количества часов).
- [ ] **Дашборд агента**:
  - Список сгенерированных видеороликов, запланированных на публикацию, с возможностью отменить или отредактировать каждый конкретный пост до его выхода.
